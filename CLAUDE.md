## Project

**PreçoPerto** — a collaborative price-comparison PWA. Users scan grocery receipts (OCR), the system extracts prices, and an interactive map shows where each product is cheapest, factoring in a **cost-benefit calculation** (savings vs. travel cost). Initial region: Ribeirão Preto - SP. UI language and API messages are in Brazilian Portuguese.

Monorepo with two independent apps: `backend/` (FastAPI) and `frontend/` (React + Vite). There is currently **no automated test suite** in the repo (only vendored library tests under `venv/`).

## Commands

### Backend (`cd backend`, activate `venv` first)
```bash
uvicorn app.main:app --reload --port 8000   # dev server → http://localhost:8000 (docs at /docs)
python scripts/seed_db.py                    # DROP + recreate all tables and seed demo data
```
There is **no migration step at startup** and Alembic is installed but not wired up — schema is created by `seed_db.py` via `Base.metadata.create_all`. Running the seed script **drops all existing data**.

### Frontend (`cd frontend`)
```bash
npm run dev       # dev server → http://localhost:3000, proxies /api to :8000
npm run build     # tsc -b && vite build  (also the typecheck gate)
npm run lint      # oxlint
npm run preview   # serve the production build
```

## Architecture

### Backend — async FastAPI + SQLAlchemy
- **Layering** (`backend/app/`): `main.py` mounts routers → `api/routes/*` (auth, products, markets, prices, receipts) → `models/*` (SQLAlchemy ORM) + `schemas/*` (Pydantic) → `services/*` (business logic) + `core/*` (config, db, security).
- **All routes are async.** Use `db: AsyncSession = Depends(get_db)` and `await db.execute(select(...))`. Follow the existing pattern in `api/routes/prices.py`.
- **All API routes are under `/api/v1/<resource>`** (prefix set in `main.py`). CORS is currently wide open (`allow_origins=["*"]`).
- **Database is dual-dialect.** Dev uses async SQLite (`aiosqlite`, `preco_perto.db`); production targets Supabase Postgres. Because of this, primary keys use the custom `GUID` `TypeDecorator` in `core/database.py` — it maps to Postgres `UUID` or `CHAR(36)` on SQLite. Use `GUID` for any new id/foreign-key column; do not import Postgres UUID types directly into models.
- **Config** (`core/config.py`) is a pydantic-settings `Settings` loaded from `backend/.env`. Required vars: `SUPABASE_URL`, `SUPABASE_KEY`, `DATABASE_URL`, `REDIS_URL`, `JWT_SECRET`. The app will not start if these are missing.
- **Auth** (`core/security.py`): bcrypt password hashing + JWT (python-jose, HS256).

### Backend services — the product's core logic (`app/services/`)
- `utils/distance.py` — Haversine distance + `calculate_cost_benefit(distance_km, price_diff)`. Encodes real Ribeirão Preto transport costs (walking free, car ~R$0.75/km round trip, bus R$4.40 flat) and returns per-mode `worth_it` flags. This cost-benefit output drives the app's central value proposition.
- `product_normalization.py` — `normalize_product()` (lowercase, strip accents, standardize PT-BR units: "1 litro"="1000ml"="1l", "1 quilo"="1000g"="1kg") and `fuzzy_match()` via rapidfuzz. Used to merge the same product reported across markets.
- `geocoding.py` — Nominatim (OpenStreetMap) lookup, requires `User-Agent: PrecoPerto/1.0`, cached in Redis (7-day TTL) with **graceful degradation if Redis is unavailable**.

The price-comparison endpoint (`GET /api/v1/prices/product/{id}?lat&lng&radius`) is the main read path: it filters prices to the last 30 days, computes distance per market, sorts cheapest-first, and attaches cost-benefit — see `api/routes/prices.py`.

### Frontend — React 19 + Vite PWA
- **All server access goes through `src/services/api.ts`** (axios instance). Base URL is `import.meta.env.VITE_API_URL` (set `frontend/.env` → `VITE_API_URL=http://localhost:8000`); in dev the Vite proxy also forwards `/api`.
- **Routing** (`src/App.tsx`): `react-router-dom` v7. The router `basename` and Vite `base` are both `/preco-perto/` for GitHub Pages — **keep them in sync** if the deploy path changes. Routes are id-driven (`/search/:productId`, `/map/:productId`).
- **Structure**: `pages/` (route screens) compose `components/` (MarketCard, PriceList, PriceMap, ProductSearch, ReceiptScanner); data fetching is wrapped in `hooks/` (`useProducts`, `usePrices`, `useGeolocation`); shared types in `src/types/`.
- **OCR is client-side** via `tesseract.js` in `ReceiptScanner.tsx` (camera → canvas → OCR); the backend `receipts` route stores results.
- **Map** uses Leaflet / react-leaflet over OpenStreetMap tiles.
- **PWA** assets are static in `frontend/public/` (`manifest.json`, `sw.js`, icons) — not generated by a plugin.

### Deploy
- Frontend deploys to **Vercel** (root dir: `frontend/`, build: `npm run build`, output: `dist`). Vercel project needs `VITE_API_URL` env var pointing to the Railway backend URL.
- Backend deploys to **Railway** via `backend/Dockerfile`. Health check at `/health`. Env vars from `.env.production.example`.
- Database: **Supabase** (PostgreSQL + PostGIS). Storage buckets: `receipts`, `offer_flyers`.
- Cache: **Upstash** (Redis REST URL).
- CI: `.github/workflows/ci.yml` runs lint + build + test on PRs.

## Methodology (Spec-Driven Development)

The project follows SDD. Authoritative planning artifacts live in `specs/001-price-comparison/` (`spec.md`, `plan.md`, `tasks.md`, `data-model.md`, `contracts/`, `clarifications.md`) and immutable principles in `.specify/memory/constitution.md`. When adding features, check `data-model.md` and `contracts/` for the intended schema and API shape before implementing. Constitutional non-negotiables to respect: mobile-first PWA, radical UX simplicity, every price must carry timestamp + source (`source` field is required on `Price`), geolocation-anchored data, and transparent cost-benefit.
